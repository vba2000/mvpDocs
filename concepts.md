# Основные понятия

## Публичные и приватные данные

- **Публичные** — не требуют входа пользователя: котировки, стаканы, публичные сделки, справочники рынков. В REST это прежде всего префикс `/market/**`. В WebSocket — потоки вида `spot/orderbook.{market_id}`, `spot/trades.{market_id}` и т.д. (без префиксов `user`, `provider.`, `rfq/`).
- **Приватные** — привязаны к аккаунту или роли: балансы, ордера, пользовательские события, RFQ для маркет-мейкера. В REST — `/spot`, `/accounts`, `/rfq`, `/aggregate-price`, `/auth/*` (кроме явно публичных), `/admin/*`, `/users/*`. В WebSocket — потоки с префиксами из конфигурации `private-stream-prefixes`: `user`, `provider.`, `rfq/`.

Подробнее о правилах доступа: [auth/README.md](auth/README.md).

## REST и WebSocket

- **REST** — запрос/ответ HTTP(S); удобно для разовых операций (создать ордер, получить баланс, историю).
- **WebSocket** — постоянное соединение; подписка на именованные **потоки** (`streams`); сервер пушит данные в оболочке сообщения (см. ниже).

## Формат ответа REST

Успешные ответы оборачиваются в общую структуру:

| Поле | Тип | Описание |
|------|-----|----------|
| `code` | number | Код результата (0 при успехе в типичном сценарии) |
| `msg` | string | Текстовое сообщение |
| `result` | object \| null | Полезная нагрузка метода (тип зависит от эндпоинта) |
| `extInfo` | object | Доп. метаданные |
| `time` | number (int64) | Время ответа (epoch ms) |

Ошибки аутентификации/авторизации на уровне фильтров могут отдаваться в том же стиле с `result: null` и полями в `extInfo` (`type`, `detail`, `instance`).

## Формат сообщений WebSocket

Исходящие от сервера сообщения описаны в [`protos/ws.proto`](protos/ws.proto): оболочка `WsEnvelope` → `WsResult` с вариантами `subscribed`, `unsubscribed`, `ping`, `data`. Поле **`data`** в событии рыночных/пользовательских данных — это сериализованный protobuf (в binary-режиме) или JSON-представление (в JSON-режиме).

Поле **`trace_id`** дублируется на нескольких уровнях — для корреляции запросов и логов.

## Роли и права (кратко)

- **`OPERATOR`** — административные REST-эндпоинты `/users/**` и ряд операций «от имени пользователя» через query `userId` там, где это указано в API.
- **`MARKET_MAKER`** — RFQ-котировки и потоки провайдера (`provider.*`, `rfq/provider`).
- Для пользовательского потока **`user`** на стороне WebSocket требуется право **`READ`** (модель `Permission` в бэкенде).

Точные проверки: WebSocket — логика `hasStreamPermission` в сервисе авторизации WS; REST — Spring Security (`authenticated`, `hasAuthority("OPERATOR")`).
