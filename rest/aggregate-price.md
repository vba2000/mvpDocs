# REST: aggregate price

## Назначение

Группа `/aggregate-price/*` возвращает агрегированные индексные цены по рынкам. Это не торговые команды, а read-only сервис для аналитики, риск-моделей, UI и внешних расчетов.

Все методы группы требуют аутентификацию по `API key`.

## Методы

| Метод | Путь | Назначение |
|------|------|------------|
| `GET` | `/aggregate-price/latest` | Последние индексные значения |
| `GET` | `/aggregate-price/history` | История индексных значений |

## `GET /aggregate-price/latest`

Query:

| Параметр | Тип | Обязателен | Назначение |
|----------|-----|------------|------------|
| `markets` | string[] | да | Список market ids, например `BTC_USDT` |

Использование:

- быстрый snapshot для UI;
- расчет reference price;
- предторговая аналитика.

Ответ содержит массив `AggregatePriceDto`:

| Поле | Тип | Назначение |
|------|-----|------------|
| `marketId` | string | Рынок |
| `aggregatePrice` | number | Агрегированная цена |
| `change24` | number | Изменение за 24 часа |
| `timestamp` | string(date-time) | Время значения |

## `GET /aggregate-price/history`

Query:

| Параметр | Тип | Обязателен | Назначение |
|----------|-----|------------|------------|
| `markets` | string[] | да | Список рынков |
| `from` | integer | да | Начало диапазона, epoch millis |
| `to` | integer | да | Конец диапазона, epoch millis |
| `interval` | string | да | `1min`, `10min`, `1h`, `1d`, `1w` |
| `limit` | integer | нет | Максимум точек на рынок |

## Бизнес-сценарии

- построение индексных графиков;
- риск-контроль;
- хранение reference price вне торгового контура;
- сверка котировок собственного pricing engine с внешним индексом.

## Отличие от market data

- `/market/*` ориентирован на торговую market data платформы;
- `/aggregate-price/*` дает агрегированную индексную цену из внешних источников.

Это два разных информационных слоя, и бизнесу нельзя подменять один другим без явного решения.
